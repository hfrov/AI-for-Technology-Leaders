# 09 - Security

## AI Expands the Attack Surface
AI systems introduce new risks across data, models, prompts, APIs, and supply chains. Security must be embedded across the full lifecycle.

## Priority Security Domains
- Data protection and access boundaries
- Model integrity and tamper resistance
- Prompt and output safety controls
- Third-party model and vendor risk

## Secure-by-Design Practices
### Identity and Access
Apply least privilege and strong authentication across AI infrastructure.

### Data Controls
Protect training and inference data with classification, encryption, and retention policies.

### Runtime Defenses
Detect abuse patterns, prompt injection, and anomalous model behavior.

### Incident Response
Extend security playbooks to include AI-specific failure modes.

## Executive Security Questions
- Which AI assets are mission-critical?
- How do we detect and contain AI misuse quickly?
- Are vendor and model dependencies continuously assessed?
